Sub-Specialization New

58 sub-specializations across 17 role families enable per-request role refinement via hybrid keyword resolution — the right contract for the right context, every time.

Rationale: A "Frontend Developer" working with React requires a fundamentally different system prompt than one working with Svelte. Sub-specs load the appropriate contract per request without bloating or duplicating the parent role definition.

Resolution Algorithm

The routing engine implements a hybrid resolution strategy: scan request for keywords: in each sub-spec's YAML frontmatter, apply layer × framework matching, and fall back to the generic parent contract when no sub-spec match is found — ensuring zero flow interruption.

  1. Keyword scanning — Agent scans request text against keywords: fields in each sub-spec's YAML frontmatter
  2. Layer × Framework matching — Match on both architectural layer and specific framework/technology for precise resolution
  3. Sub-spec loading — Exact match loads sub-spec contract.md + workflow.md; parent role cached as fallback
  4. Graceful fallback — No match resolves to generic parent contract (never breaks execution flow)

Full algorithm documented in .crewmarkdown/procedures/role-subspec.md

Debate Framework

Structured decision-making framework for resolving disagreements when multiple viable options exist. Ensures all perspectives are evaluated on merit with documented rationale.

Cross-cutting: Can be activated at decision points within any lifecycle objective.

Purpose

When multiple valid approaches exist for a decision, the debate framework ensures all perspectives are systematically evaluated, trade-offs are explicitly documented, and the optimal option is selected based on agreed scoring criteria rather than bias or recency.

Process Flow

1. Debate Trigger → 2. Panel Assembly → 3. Opening Statements → 4. Cross-Examination → 5. Scoring → 6. Resolution
  1. Debate Trigger — A decision point is identified with multiple viable options. The objective lead calls for a structured debate.
  2. Panel Assembly — Facilitator assembles roles with a stake in the decision. Each panelist represents their domain perspective.
  3. Opening Statements — Each panelist presents their position with supporting evidence and reasoning.
  4. Cross-Examination — Structured rounds of rebuttal, questioning, and evidence presentation.
  5. Scoring — Panelists evaluate options against agreed criteria using a weighted scoring system. Highest aggregate score wins.
  6. Resolution — Decision is formally recorded with rationale, rejected alternatives, and expected consequences.

Meeting Mode

Dynamic real-time meetings with intelligent role assembly, structured brainstorming, formal decision-making, and actionable outcome tracking.

Cross-cutting: Activated via /meeting [topic] at any point in the lifecycle for ad-hoc or scheduled collaboration.

Execution Schedule

Facilitator: Convene → Agenda → Brainstorm → Decisions → Action Items → Minutes

Dynamic Role Assembly

Topic keywords in the meeting request determine which roles are automatically assembled. The facilitator dynamically adjusts the roster based on the conversation's evolution:
