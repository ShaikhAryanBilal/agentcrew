Why CrewMarkdown?

AI agents are exceptionally capable at writing code, but they lack structured software engineering process. Without a defined framework, every prompt produces inconsistent results — code arrives without tests, threat models, PRDs, or deployment plans. The result is unpredictable quality and costly rework.

CrewMarkdown solves this by providing any AI agent with a repeatable, production-grade engineering framework. 35 specialized roles + 78 sub-specializations, each with hardened contracts, structured workflows, and quality gates. Drop it into any project, point your AI at it, and get consistent, professional results.

No SaaS. No API keys. No vendor lock. Just plain markdown files that any AI agent can read and execute against.

How It Works

User Request → Skill-Level Inference → Objective Match → Squad Assembly → Execute & Verify → Ship ✓

When you make a request, the routing engine first infers your skill level from your input (specificity, jargon, role mentions), then matches your intent against a YAML-defined routing table, identifies the correct lifecycle objective, assembles the appropriate squad of roles, and executes the workflow with built-in quality gates at every phase transition.

Skill-Level Inference (Auto/Guided/Expert)

The agent reads your input and adjusts autonomy automatically — no configuration needed.

Auto
"Build me a landing page"
Agent handles everything, user approves final output
Guided
"What stack should I use?"
Agent proposes options, user decides at key junctions
Expert
"Architect, design the DB schema"
Agent executes precisely what's asked with minimal interpretation

Execution Modes

Solo
"Architect, design the DB"
Single role executes independently with zero orchestration overhead
Squad
"Squad (PM, BA), clarify requirements"
Multiple roles collaborate on one objective with parallel execution paths
Orchestrate
"Build the payment system"
Orchestrator decomposes into objectives, assigns squads, tracks progress, coordinates delivery
Meeting
"/meeting brainstorm [topic]"
Dynamic role assembly with structured deliberation, decisions, and actionable outcomes

Agent-Centric Format

Every file in CrewMarkdown is designed as a system prompt for an AI agent — YAML metadata for rapid parsing, structured markdown for direct reasoning, and code-block templates for drop-in artifact generation.

Section Format Agent Benefit
Needs/Produces, Skills, Rules YAML frontmatter Parse in 1ms, schema-enforced
System, Instructions, I/O, Rules Structured markdown No prose parsing — direct to reasoning
Templates Code blocks Drop-in artifact generation

Zero dependencies. Every file is a self-contained prompt.

The Crew 35 Roles 78 Sub-Specs

35 invocable AI roles spanning engineering, product, design, data, ML, quality, security, operations, business, governance, and collaboration. Each role ships with a YAML-frontmatter contract, structured workflow, quality checklist, and artifact templates. 19 role families offer 78 sub-specializations for framework-specific prompt refinement.

Quick Start

Zero-Config (for everyone)

# 1. Copy to your project
cp -r .crewmarkdown ./your-project/

# 2. Run the entry point
./go.ps1

Type what you need in plain English. No docs, no setup. The agent handles the rest.

Power User Setup

# 1. Copy to your project
cp -r .crewmarkdown ./your-project/

# 2. Generate codebase map
./.crewmarkdown/scripts/generate-codebase-map.ps1

# 3. Configure AI agents
./.crewmarkdown/config/setup.ps1        # Windows
./.crewmarkdown/config/setup.sh --all   # Linux/macOS

# 4. Make a request
#    "PM, write a PRD for login"
#    "Squad (QA, Security), verify quality"
#    "Build the full e-commerce platform"

Your AI agent reads .crewmarkdown/00-objectives.md, routes your request to the correct objective using the YAML-defined routing table, loads the appropriate squad contracts and workflows, and executes with built-in quality gates.
