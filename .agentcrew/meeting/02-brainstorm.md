# Meeting: Brainstorm — Ideation + Discussion

## Need
- Agenda set with topic + timeboxes
- Roles assembled and present

## Instructions

### 1. Opening Context
1. Facilitator presents: what are we solving? why does it matter? constraints? desired outcome?

### 2. Round-Robin Ideation
1. Each role contributes from their perspective, one at a time:
   - PM: Business goals, user needs, priorities
   - UX: User flows, design considerations, personas
   - Frontend: UI approach, component considerations
   - Backend: API needs, data model impact
   - Tech Lead: Technical constraints, patterns, risks

### 3. Silent Ideation (Optional)
1. For complex topics, give each role 2 min to write ideas independently first, then share

### 4. Parking Lot
1. Off-topic but valuable ideas go to the parking lot

### 5. Summary
1. Facilitator summarizes all ideas gathered
2. Group into themes

## Techniques
- **Round-Robin**: Each role speaks in turn. No one speaks twice until everyone has spoken once.
- **Brainwriting**: Roles write ideas silently, then pass to next role to build upon.
- **Yes And**: Build on ideas before critiquing. "Yes, and we could also..."
- **SCAMPER**: Substitute, Combine, Adapt, Modify, Put to other use, Eliminate, Rearrange

## Anti-Patterns
| Don't | Instead |
|-------|---------|
| Evaluate during ideation | Capture first, evaluate later |
| Let one person dominate | Enforce round-robin |
| Dismiss ideas quickly | Park them if not relevant now |
| Skip quiet roles | Specifically invite them |
