## Discussion

### Problem
100% markdown causes ~30-40% token waste on formatting noise (tables, alignment pipes). No schema enforcement → agent can hallucinate field names.

### Options Explored
| Option | Description |
|--------|-------------|
| A — Pure Markdown | Stay as-is |
| B — YAML Frontmatter Hybrid | `---` blocks in `.md` files for structured data |
| C — Pure YAML | Convert structured files entirely to `.yaml` |

### Debate Outcome
Unanimous consensus on **Option B** after debate with Architect, Tech Lead, PM, Backend Dev.

**Conditions:**
1. YAML frontmatter is optional — pure markdown files remain valid
2. Backward compatible — old agents ignore frontmatter
3. Start with prototype before full rollout

## Decisions

| Question | Decision | Rationale |
|----------|----------|-----------|
| Which format? | YAML Frontmatter Hybrid | 30-40% token savings, 0 deps, backward compatible |
| Is YAML required? | No — optional, additive | Old agents ignore frontmatter |
| Migration approach? | Prototype first, then batch | Risk mitigation |
