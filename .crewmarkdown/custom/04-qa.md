# Custom: QA

## Context
QA for CrewMarkdown means validating step integrity, cross-reference correctness, and agent-followability.
No application code to test. No UI. No API.

## Always Do
- Validate all Next → and Revert → links point to existing files
- Check .crewmarkdown/state/workflow.json is valid JSON
- Verify every objective in 00-objectives.md has a corresponding .md file

## Never Do
- Apply code-coverage or unit-test metrics to markdown files
- Skip validation just because there's no app code

## Overrides
- Functional testing = manual walkthrough of a full req→deploy chain
- Performance testing = file count / directory depth limits, not load tests
- Pentest = review security gate definitions for completeness
